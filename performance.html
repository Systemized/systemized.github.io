Performance Page

This page describes the performance optimizations applied to the website. The goal is to reduce how long the browser waits before it can display the page, and to keep the size of the response small.

CDN (Content Delivery Network)

GitHub Pages serves traffic through the Fastly CDN. When the browser resolves systemized.github.io, the IP addresses returned belong to Fastly edge servers that cache the static files in many locations. A user in a different region downloads the page from a nearby edge server instead of from GitHub's origin, which lowers latency.

The CDN behavior is visible in the response headers: 

$ curl -I https://systemized.github.io

HTTP/2 200
server: GitHub.com
content-type: text/html; charset=utf-8
cache-control: max-age=600
age: 42
x-served-by: cache-iad-kiad7000051-IAD
x-cache: HIT
x-cache-hits: 1

The x-cache: HIT line shows that the file was returned from the edge cache rather than re-fetched from origin. The cache-control header lets the browser keep its own copy for ten minutes, which avoids a network request entirely on repeat visits within that window.

Browser Caching

Because every page links to the same index.css file, the browser only downloads the stylesheet on the first request and reuses its cached copy on every page after that. The cache-control: max-age=600 header sent by GitHub Pages also lets the browser keep that copy across page reloads for ten minutes without re-validating with the server.

Lazy Loading

Any image added to the site uses the native loading="lazy" attribute: 

<img src="diagram.png" alt="network diagram" loading="lazy" width="600" height="400">

With this attribute the browser only downloads the image when it is about to enter the viewport. Specifying width and height reserves layout space ahead of time so the page does not shift when the image arrives.

Preconnect and Resource Hints

For any external origin the page might contact later (for example a font CDN or an analytics endpoint), a preconnect hint can be added in the <head>: 

<link rel="preconnect" href="https://www.google-analytics.com">

This lets the browser perform the DNS lookup and TLS handshake in parallel with parsing the HTML, so the actual request later finishes faster.

The CDN and browser caching are the two changes that have the largest visible effect on page load. Lazy loading and preconnect hints become more useful as the site grows to include images and external resources.
