Monitoring Page

This page describes how traffic to the website is monitored, and includes a small client-side counter that tracks how many times this browser has loaded the page.

GitHub Pages Traffic Insights

GitHub records request counts for repositories that publish a Pages site. The traffic dashboard at github.com/Systemized/systemized.github.io/graphs/traffic shows visitors and page views over the last 14 days, broken down by referring site and by path. This is server-side data, so it is not affected by ad blockers or client-side script filters.

Client-Side Visit Counter

Server-Side Analytics

For aggregated visitor data, a tag such as Google Analytics can be added. The installation requires loading the gtag.js script and updating the Content Security Policy to allow it. The relevant additions look like this: 

<link rel="preconnect" href="https://www.googletagmanager.com">
<script async src="https://www.googletagmanager.com/gtag/js?id=G-XXXXXXXXXX"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'G-XXXXXXXXXX');
</script>

This is documented here rather than enabled, since the current site does not need a real measurement ID and adding the script would require loosening the CSP.

Network-Level Inspection

For ad-hoc inspection of the requests a browser makes when loading the site, Wireshark can capture the TLS handshake and the HTTP/2 request headers, and the browser's developer tools (Network tab) show the same data without a packet capture. Both are useful for verifying that the security and cache headers described on the other pages are actually being sent by the server.

GitHub's traffic graph covers aggregate usage, the local counter confirms script execution on the client, and Wireshark or DevTools cover the per-request detail.
